Methodology & confidence

This is a modelled sequence, stress-tested — not an executable schedule

The file has no per-cohort module dates (module dates on the Modules tab are min-start / max-end aggregated across all ~9 cohorts on each programme, so they can't be attributed to a specific cohort). The sequence is therefore anchored on the only genuine per-cohort facts — each cohort's real programme start and end — with a 4-month term model laid on top (T1/T2/T3, evidenced by the T2/T3 formative-submission modules in the data). Transition = first term boundary after the fence, only if a real term remains (else the cohort just finishes on Totara).

Stress test against the one clean cohort (Software Leadership +PDP — single cohort, so its module dates are un-aggregated actuals):

  • Taught phase (where every transition in this plan falls): ±~1 month. Modelled T1→T2 boundary landed 17 days from the real one; the other two within ~5 weeks, inside the real inter-term gaps. Actual data confirmed ~10-week teaching blocks ≈ 4-month terms.
  • Dissertation / PDP tail: unreliable (up to ~23 weeks off) — the back ~6 months of a +PDP programme is one long undated project with no term structure. Does not affect the migration decision: fence-crossings happen in the taught phase, and once a cohort has crossed it's on new Learn regardless.

Module-level view: module names and sequence are real (from the module registrations); the term→module grouping is derived from that real ordering (earliest-start buckets, cross-checked against the clean cohort); only the dates against them are modelled. The term→module map is the one extra assumption that view adds — flag it for SoT validation alongside the dates.

Confidence: good enough for planning wave shape, scope and readiness (±~1 month in the taught phase). Evidence base is one clean cohort (18-month PDP variant), so treat "±1 month" as a working estimate, not a validated spec for the dominant 12-month Cyber structure. Do not hang an individual student's move date on it.

Switch trigger: the moment School of Tech confirms a file with real per-module start/end dates per cohort, drop the modelled terms and use those. Until then, this model is the recommended basis (there are no true per-cohort module dates in this file to use instead).
